JavaScript

JavaScript es un lenguaje de programación interpretado, de alto nivel y orientado a objetos. Es uno de los tres pilares fundamentales del desarrollo web, junto con HTML y CSS. Mientras que HTML estructura el contenido y CSS lo estiliza, JavaScript agrega interactividad y dinamismo a las páginas web.

JavaScript fue creado por Brendan Eich mientras trabajaba en Netscape Communications Corporation. Inicialmente, el lenguaje fue desarrollado en solo 10 días bajo el nombre de Mocha, pero más tarde se cambió a LiveScript. Finalmente, fue rebautizado como JavaScript poco antes de su lanzamiento en el navegador Netscape Navigator 2.0 en diciembre de 1995. Aunque su nombre sugiere una relación con Java, JavaScript es un lenguaje completamente diferente. La decisión de usar el nombre JavaScript fue en parte una estrategia de marketing para capitalizar la popularidad de Java en ese momento.

Como se puede agregar a HTML?

Etiqueta <script> dentro del archivo HTML.

Puedes escribir código JavaScript dentro de las etiquetas <script> en tu archivo HTML.
Es facil de agregar y de manejar en el archivo HTML.

  • Coloca el código JavaScript dentro de una etiqueta <script>.

  • Inserta la etiqueta <script> en el <head> o en el body, dependiendo de cuando quieres ejecutar el script.

  • Ahora solo necesitas guardar el archivo y agregar el codigo al script.

Etiqueta <script> en archivo separado.

Es recomendable colocar el código JavaScript en un archivo separado con extensión .js y enlazarlo en tu documento HTML usando la etiqueta <script> con el atributo src.

  • Crea un archivo nuevo y guárdalo con la extensión .js. Por ejemplo: script.js.

  • Dentro del archivo script.js, escribe el código JavaScript que deseas utilizar.

  • Si ya tienes un archivo HTML, ábrelo. Si no, crea uno nuevo.

  • Dentro del archivo HTML, coloca la etiqueta <script> con el atributo src apuntando al archivo JavaScript. Esta etiqueta puede ir dentro del <head> o en el <body>.

Variables

Las variables en JavaScript son contenedores que almacenan datos. Permiten a los programadores guardar, modificar y manipular valores durante la ejecución de un programa. Existen tres formas principales de declarar variables en JavaScript, utilizando las palabras clave var, let y const. Cada una tiene características y reglas de uso que afectan su alcance, mutabilidad y comportamiento.

  • var: es una forma de declarar variables en JavaScript que tiene alcance de función, lo que significa que una variable declarada con var es accesible en toda la función en la que se encuentra, o globalmente si está fuera de cualquier función.
  • let: es una forma de declarar variables en JavaScript con alcance de bloque, lo que significa que solo es accesible dentro del bloque de código {} donde se declara, como en una función, un bucle o una estructura condicional.
  • const: es una forma de declarar variables en JavaScript que, al igual que let, tiene un alcance de bloque, es decir, solo es accesible dentro del bloque {} donde se declara. La principal característica de const es que no permite la reasignación: una vez que se asigna un valor a una variable declarada con const, no puede cambiar.

Los operadores aritméticos permiten realizar cálculos matemáticos:

  • Suma (+): Añade dos operandos.

  • Resta (-): Resta el segundo operando del primero.

  • Multiplicación (*): Multiplica dos operandos.

  • División (/): Divide el primer operando por el segundo.

  • Módulo (%): Devuelve el resto de dividir el primer operando por el segundo.

  • Exponenciación (**): Eleva el primer operando a la potencia del segundo.

Funciones

Las funciones en JavaScript son bloques de código reutilizable que realizan una tarea específica o calculan un valor. Se pueden declarar con la palabra clave function o mediante funciones de flecha (=> en ES6) y se ejecutan cuando se invocan (llaman).

Las funciones pueden recibir parámetros como entrada y devolver un valor como resultado con la palabra clave return. Existen dos tipos principales: funciones declaradas, que pueden ser invocadas en cualquier parte del código debido al "hoisting", y funciones expresadas, que se asignan a una variable y solo pueden ser llamadas después de su definición. Las funciones permiten modular el código, haciéndolo más legible y reutilizable.

Datos Principales

Los tipos de datos principales son fundamentales para el manejo y manipulación de información en JavaScript.

  • String: Representa una secuencia de caracteres, como palabras o frases. Se define entre comillas simples ('), dobles ("). Ejemplo: "Hola Mundo", 'JavaScript'.

  • Number: Representa tanto números enteros como decimales. JavaScript no distingue entre tipos como enteros y flotantes, todo se maneja como number. Ejemplo: 42, 3.14.

  • Booleano: Solo tiene dos posibles valores: true o false. Se utiliza para representar condiciones lógicas, comparaciones y en estructuras de control como if. Ejemplo: true, false.

Eventos

Los eventos en JavaScript son acciones o sucesos que ocurren en la página web y que pueden ser detectados y respondidos por el lenguaje. Los eventos permiten a los desarrolladores crear interactividad al reaccionar a cosas como clics del usuario, presionar teclas, cargar la página, enviar formularios, y más. Cada evento está asociado con un elemento del DOM (Document Object Model), y se pueden manejar mediante event listeners (escuchadores de eventos) que ejecutan funciones específicas cuando ocurre el evento.

  • click: Ocurre cuando el usuario hace clic en un elemento.
  • keydown:Se activa cuando una tecla es presionada.
  • submit: Se dispara cuando un formulario es enviado.
  • mouseover:Ocurre cuando el cursor pasa por encima de un elemento.
  • load: Se activa cuando una página o recurso ha terminado de cargarse.

DOM

El DOM es una interfaz de programación para los documentos HTML y XML. Representa la estructura de la página web como un árbol de nodos, donde cada nodo es un elemento (etiqueta), atributo o contenido textual. Esto permite a los lenguajes de programación, como JavaScript, interactuar con el documento, modificar su estructura, estilo o contenido.

El HTML DOM es una representación jerárquica del contenido de un documento HTML. Cada parte del HTML, como etiquetas, atributos y contenido de texto, es representada como un objeto en el árbol del DOM. Esto permite que JavaScript acceda y manipule esos objetos fácilmente.

¿Para qué sirve el DOM?

Etiqueta <script> dentro del archivo HTML.

El DOM sirve para manipular dinámicamente el contenido de una página web.
A través de JavaScript, los desarrolladores pueden:

  • Modificar la estructura de la página (agregar, eliminar o modificar elementos).
  • Cambiar estilos CSS dinámicamente.
  • Manejar eventos (como clics, desplazamientos o entradas de teclado).
  • Acceder a los atributos de los elementos HTML y modificar su valor.

Conversión en Tokens (Tokenization)

La conversión a tokens en el DOM (Document Object Model) es un proceso fundamental en la manipulación de contenido dinámico en las aplicaciones web modernas. En términos simples, los "tokens" son las unidades mínimas de información que un sistema de análisis sintáctico utiliza para interpretar y manipular el contenido del documento HTML. Al trabajar con el DOM, cada nodo (como elementos, atributos o texto) puede ser considerado un token que el navegador convierte y estructura en un árbol de objetos que representa la jerarquía del documento.

Esta conversión permite que los desarrolladores interactúen con estos objetos a través de JavaScript, modificando el contenido y estructura del documento de forma dinámica y eficiente.

Lexicalización

La lexicalización en el contexto del DOM y el procesamiento de contenido web se refiere al proceso mediante el cual el texto sin formato es descompuesto en unidades semánticamente significativas llamadas "tokens". Este es un paso clave en la interpretación del código fuente HTML, CSS o JavaScript antes de que el navegador pueda construir el árbol DOM y renderizar la página. La lexicalización se lleva a cabo por un analizador léxico o "lexer", que escanea el flujo de caracteres y los agrupa en tokens. Estos tokens pueden representar elementos HTML, atributos, operadores, palabras clave, nombres de variables, entre otros.

Construcción (Tree Construction)

La construcción del árbol o tree construction en el contexto del DOM es el proceso mediante el cual el navegador toma el contenido de un documento HTML y lo convierte en una estructura jerárquica llamada árbol DOM (Document Object Model). Este proceso sigue a la lexicalización y el análisis sintáctico, en los que se convierten los caracteres y el texto del documento en tokens, y luego se interpretan estos tokens para generar la estructura que describe cómo están organizados los elementos del documento.

getElementById()

El método getElementById() es una de las herramientas más utilizadas en la manipulación del DOM, ya que permite seleccionar un elemento específico del árbol DOM mediante su atributo id. Este método es particularmente eficiente porque el identificador id debe ser único en todo el documento, lo que garantiza que siempre se seleccionará un único elemento.

Al invocar document.getElementById('idDelElemento'), el navegador busca rápidamente en el árbol DOM hasta encontrar el elemento que tiene el valor de id especificado. Una vez que se obtiene el elemento, este se puede manipular de diversas maneras, como modificar su contenido, agregar o eliminar clases, cambiar atributos o incluso aplicarle estilos directamente.

getElementsByClassName()

El método getElementsByClassName() es una poderosa herramienta para seleccionar múltiples elementos del DOM que comparten la misma clase. A diferencia de getElementById(), que solo devuelve un único elemento, este método retorna una colección de todos los elementos que tienen una o más clases específicas.

Al invocar document.getElementsByClassName('nombreDeLaClase'), el navegador escanea el árbol DOM en busca de todos los elementos cuyo atributo class contenga el valor proporcionado. Esta colección de elementos devuelta no es un arreglo estándar, sino una HTMLCollection, que es una lista en vivo. Esto significa que cualquier cambio en el DOM afectará automáticamente a esta colección.

getElementsByTagName()

El método getElementsByTagName() es una herramienta fundamental para seleccionar todos los elementos de un documento HTML que tienen un determinado nombre de etiqueta. Este método devuelve una colección de nodos que coincide con el nombre de la etiqueta proporcionada, permitiendo manipular todos los elementos de un tipo específico en el DOM.

Por ejemplo, al invocar document.getElementsByTagName('div'), el navegador recopila todos los elementos <div> presentes en el documento y los devuelve en una HTMLCollection. Esta colección es en vivo, lo que significa que cualquier modificación que se haga en el DOM (como añadir o eliminar elementos de ese tipo) se verá reflejada automáticamente en la colección.

querySelector()

El método querySelector() es una herramienta poderosa y versátil en el API del DOM que permite seleccionar el primer elemento que coincide con un selector CSS específico dentro del documento. Al invocar document.querySelector('selector'), se puede utilizar cualquier selector válido de CSS para localizar un elemento, lo que incluye selectores de clases, identificadores, atributos, combinaciones de estos y pseudo-clases.

Por ejemplo, document.querySelector('#miId') seleccionará el primer elemento con el id miId, mientras que document.querySelector('.miClase') seleccionará el primer elemento con la clase miClase.

querySelectorAll()

El método querySelectorAll() es una herramienta altamente flexible y potente en el API del DOM que permite seleccionar todos los elementos en un documento que coinciden con un selector CSS específico. Al invocar document.querySelectorAll('selector'), se puede utilizar cualquier selector CSS válido para seleccionar múltiples elementos al mismo tiempo, lo que incluye selectores de clase, identificadores, atributos, combinaciones de estos y pseudo-clases.

Por ejemplo, document.querySelectorAll('.miClase') seleccionará todos los elementos con la clase miClase, y document.querySelectorAll('div > p.intro') seleccionará todos los párrafos con la clase intro que son hijos directos de un div.

Atributo Value

El atributo value es un atributo fundamental en HTML que se utiliza para definir el valor de varios tipos de elementos de formulario, como <input>, <select> y <textarea>. Este atributo es crucial para capturar y enviar datos de los formularios al servidor, así como para interactuar con los usuarios en aplicaciones web.

Para los elementos <input>, el atributo value especifica el valor que se envía con el formulario cuando el usuario lo envía. Por ejemplo, en un campo de texto, el atributo value puede ser utilizado para establecer el texto que aparece por defecto en el campo. If else

El if-else es una estructura fundamental en la programación que permite la ejecución condicional de bloques de código basado en el resultado de una expresión booleana. En términos sencillos, el if-else se utiliza para tomar decisiones dentro del flujo de ejecución de un programa.

La estructura básica de un bloque if-else comienza con una declaración if, seguida de una condición entre paréntesis. Esta condición es una expresión que se evalúa como verdadera (true) o falsa (false). Si la condición evaluada es verdadera, se ejecuta el bloque de código que sigue a la declaración if. Por otro lado, si la condición es falsa, se omite este bloque de código y se puede ejecutar un bloque alternativo de código que sigue a la declaración else.

Propiedad Style

JLa propiedad style en el DOM de JavaScript proporciona una forma directa de manipular los estilos CSS de un elemento HTML desde el código. Esta propiedad es parte de la interfaz HTMLElement y permite acceder y modificar los estilos en línea de un elemento de manera programática. Al interactuar con la propiedad style, puedes ajustar varias características de presentación del elemento, como color, tamaño, márgenes, fuentes y mucho más.

Uso Básico

La propiedad style se accede a través de un objeto que representa los estilos en línea del elemento. Puedes asignar valores a propiedades CSS específicas usando la notación de punto o la notación de corchetes. La notación de punto se utiliza para establecer propiedades individuales, mientras que la notación de corchetes es útil cuando el nombre de la propiedad incluye caracteres especiales o espacios. La propiedad style permite modificar cualquier propiedad CSS que se pueda aplicar en línea. Algunos ejemplos:

  • color: Cambia el color del texto.
  • backgroundColor: Establece el color de fondo del elemento.
  • fontSize: Ajusta el tamaño de la fuente.
  • margin: Define los márgenes alrededor del elemento.
  • padding: Establece el espacio interior entre el borde del elemento y su contenido.
  • border: Configura el estilo, ancho y color del borde.

ClassList

La propiedad classList en JavaScript proporciona métodos para manipular las clases de un elemento HTML de manera eficiente. Estos métodos incluyen:

  • add(className): Añade una o más clases al elemento. Si la clase ya está presente, no se agrega de nuevo.

  • remove(className): Elimina una o más clases del elemento. Si la clase no está presente, no se realiza ninguna acción.

  • toggle(className, [force]): Alterna la presencia de una clase en el elemento. Si la clase está presente, se elimina; si no está presente, se agrega. El segundo argumento opcional force permite forzar la adición (true) o la eliminación (false) de la clase.

  • contains(className): Verifica si el elemento tiene una clase específica. Devuelve true si la clase está presente y false si no lo está.

  • replace(oldClass, newClass): Reemplaza una clase existente por una nueva clase. Si oldClass no está presente, no se realiza ninguna acción.

  • item(index): Devuelve la clase en la posición especificada por el índice dado. El índice es cero basado, por lo que el primer elemento tiene un índice de 0.

  • forEach(callback): Ejecuta una función proporcionada una vez por cada clase en el classList, en el orden en que aparecen en la lista. La función de callback recibe cada nombre de clase como argumento.

  • entries(): Devuelve un objeto Array Iterator que contiene pares clave-valor de la forma [index, className], permitiendo iterar sobre las clases y sus índices en el classList.

    • Estos métodos proporcionan una interfaz robusta y fácil de usar para gestionar las clases CSS de los elementos en el DOM, facilitando la manipulación dinámica de estilos y la interacción con el contenido web.

    Eventos

    En la programación web, los eventos son acciones o sucesos que ocurren en el navegador, como un clic del usuario, el envío de un formulario o la carga de una página. Los eventos son una parte crucial de la interacción con los usuarios y permiten que las aplicaciones web respondan a diferentes acciones del usuario o cambios en el estado de la aplicación.

    • Eventos del mouse: click, dblclick, mouseover, mouseout
    • Eventos del teclado: keydown, keyup, keypress
    • Eventos de formulario: submit, change, focus, blur
    • Eventos de carga: load, DOMContentLoaded
    • Eventos de red: error, abort

    Etiqueta Viewport

    La etiqueta <meta name="viewport"> es un elemento crucial en el desarrollo web moderno, especialmente cuando se trata de garantizar que las aplicaciones y sitios web sean accesibles y funcionales en una amplia gama de dispositivos, desde teléfonos móviles hasta pantallas de escritorio. Esta etiqueta se coloca en el <head>del documento HTML y juega un papel fundamental en el diseño adaptable y en la optimización de la experiencia del usuario en dispositivos móviles.

    La principal función de la etiqueta <meta name="viewport"> es controlar el tamaño y la escala del área de visualización de una página web. El área de visualización, o viewport, es la región visible del documento que el navegador muestra al usuario. Sin una configuración adecuada de la etiqueta viewport, los sitios web pueden no mostrarse correctamente en dispositivos móviles, con problemas como texto demasiado pequeño para leer o elementos que se desbordan de la pantalla.

    La etiqueta <meta name="viewport">es esencial para el diseño adaptativo (responsive design), una técnica que permite a las páginas web adaptarse a diferentes tamaños de pantalla y resoluciones. Sin una configuración adecuada de viewport, los sitios web pueden no ofrecer una experiencia óptima en dispositivos móviles, lo que puede resultar en un aumento en la tasa de rebote y una disminución en la satisfacción del usuario.

    Al implementar correctamente la etiqueta viewport, los desarrolladores aseguran que sus sitios web se visualicen de manera efectiva en cualquier dispositivo, proporcionando una experiencia de usuario consistente y accesible. Esto es especialmente importante en la era de los dispositivos móviles, donde un gran porcentaje de usuarios accede a la web desde teléfonos y tabletas.

    Media Queries

    Las media queries son una característica fundamental en CSS para crear diseños adaptativos y responsivos que se ajustan a diferentes tamaños de pantalla, resoluciones y características del dispositivo. Introducidas en CSS2 y ampliadas en CSS3, las media queries permiten a los desarrolladores aplicar estilos específicos basados en las condiciones del entorno en el que se visualiza una página web, como el ancho de la pantalla, la orientación y la resolución del dispositivo.

    La sintaxis básica de una media query incluye la palabra clave @media, seguida de una o más condiciones entre paréntesis y un bloque de estilos CSS que se aplicará si las condiciones se cumplen.

    Las media queries son esenciales para el diseño web adaptativo, que busca proporcionar una experiencia de usuario óptima en una variedad de dispositivos. Permiten a los desarrolladores diseñar interfaces que se ajusten de manera flexible y dinámica a las características del dispositivo en uso, mejorando la accesibilidad y la usabilidad. Al usar media queries, se puede garantizar que el contenido sea legible y funcional en cualquier tamaño de pantalla, desde dispositivos móviles hasta grandes pantallas de escritorio.
